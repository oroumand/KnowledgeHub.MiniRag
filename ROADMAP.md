# نقشه راه (Roadmap)

در این فایل، مسیر توسعه پروژه KnowledgeHub.MiniRag در نسخه‌های آینده مشخص شده است.

---

# نسخه 1 (وضعیت فعلی)

## قابلیت‌های پیاده‌سازی‌شده

- ثبت سند
- تقسیم متن (Chunking)
- تولید embedding
- ذخیره در Qdrant
- جستجوی معنایی (Semantic Search)
- چت مبتنی بر RAG
- مشاهده سند و chunkها
- مستندات معماری و RAG
- ثبت تصمیم‌های معماری (ADR)

---

# نسخه 1.1 — بهبود تجربه توسعه

هدف: تمیزتر شدن ساختار و بهبود تجربه توسعه‌دهنده

- انتقال queryهای read از API به Application
- بهبود logging
- بهبود error handling
- اضافه کردن نمونه درخواست‌های HTTP
- بهبود README

---

# نسخه 1.2 — مدیریت داده

هدف: کنترل بهتر داده‌ها در سیستم

- حذف سند
- حذف vectorهای مربوط به سند
- re-index کردن سند
- نمایش وضعیت index شدن
- جلوگیری از ثبت داده تکراری

---

# نسخه 2.0 — ورودی‌های پیشرفته

هدف: ورود داده از منابع واقعی

- دریافت فایل PDF
- دریافت URL
- استخراج متن از منابع مختلف
- بهبود metadata

---

# نسخه 2.1 — بهبود چت

هدف: تجربه بهتر کاربر در چت

- اضافه کردن chat history
- مدیریت session
- بهبود prompt
- نمایش بهتر منابع (citations)

---

# نسخه 2.2 — بهبود زیرساخت

هدف: نزدیک شدن به معماری production

- background indexing
- retry strategy
- health check
- بهبود configuration

---

# نسخه 3.0 — کیفیت و تست

هدف: آماده‌سازی برای توسعه تیمی

- unit test
- integration test
- CI pipeline
- بهبود deployment

---

# جمع‌بندی

این roadmap به صورت incremental طراحی شده است تا:

- توسعه مرحله‌ای باشد
- پیچیدگی کنترل شود
- پروژه قابل یادگیری باقی بماند
